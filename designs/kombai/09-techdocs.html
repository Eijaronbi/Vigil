Architecture

How Vigil processes messages from ingestion to on-chain proof.

01

Connect

OAuth-authenticated connections establish persistent WebSocket listeners on each messaging platform. Auto-reconnect with exponential backoff.
02

Classify

Messages pass through the AI classifier — urgency scoring, spam detection, sender reputation. Each message receives a priority label: Low, Medium, High, or Critical.
03

Alert

High and Critical priority messages trigger the voice alert system. The "check messages" voice command triggers a queue readout of unread priority items.
04

Attest

Critical messages are hashed (SHA-256) and attested on the Monad testnet. Each attestation includes timestamp, source, and message fingerprint.

Source Integrations

Platforms currently supported by Vigil.

WhatsApp v1

Personal and group chat monitoring via WebSocket. Encrypted session handling with persistent sync.

Telegram v2

Full MTProto API — channels, supergroups, and private conversations. Real-time update streaming.

Twitter / X v1

DMs, mentions, keyword-based tracking, and list monitoring via the streaming API.

Gmail v1

PubSub push notifications for priority inbox. Custom label routing and sender-based priority rules.
